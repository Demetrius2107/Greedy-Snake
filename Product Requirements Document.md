## 第一阶段：核心基础功能

1. 游戏窗口搭建: 创建固定大小的主窗口，设置标题、关闭规则；用画板（JPanel）承载所有游戏绘制	
2. 蛇的绘制与移动	蛇由矩形节点组成（用ArrayList存储）；自动向前移动（头加新节点，尾删节点）	
3. 方向控制	键盘↑↓←←控制方向；禁止直接反向（如向上时不能直接按向下，会直接撞死）
4. 食物系统	随机生成食物；食物不能出界、不能生成在蛇身上	随机数、碰撞检测（位置判断）
5. 吃食物逻辑	蛇头碰到食物 → 长度 + 1、分数 + 1、不删除尾部、重新生成食物	精准碰撞判断、分数逻辑
6. 游戏结束判定	两种死亡条件：①撞墙 ②撞到自己身体	边界判断、自碰撞检测
7. 分数实时显示	在界面顶部绘制当前分数	GUI 文本绘制、数据实时更新


## 第二阶段：进阶功能（锻炼交互 & 状态管理）
1. 游戏状态管理	支持：开始 / 暂停 / 运行 / 结束 4 种状态；按空格开始 / 暂停，按 R 重新开始	枚举使用、状态切换逻辑、多场景控制
2. 难度递增	分数越高，蛇移动速度越快（缩短定时器延迟）	动态参数调整、定时器优化
3. 穿墙模式	可选：撞墙后从对面边界出来（替代直接死亡）	边界逻辑优化、条件分支
4. 界面提示文字	开始界面：「按空格开始」；暂停界面：「已暂停」；结束界面：「游戏结束」	状态对应 UI 绘制、用户体验优化
5. 基础音效	吃食物、死亡、移动添加音效（Java 原生音频）	Java 音频 API、事件触发反馈

## 第三阶段：代码优化 & 规范（锻炼架构 & 可读性）
   不要把所有代码写在一个类里，拆分职责：
   Main：程序入口，创建窗口
   GamePanel：游戏核心画板，负责绘制、监听、逻辑
   SnakeNode：蛇节点类（封装 x/y 坐标，面向对象）
   Constant：常量类（存窗口大小、格子大小、颜色、速度，杜绝硬编码）
   绘制优化
   蛇头 / 身体 / 食物用不同颜色区分，背景美化，解决界面闪烁（Swing 双缓冲）
   交互优化
   防止按键过快导致方向错乱，添加方向锁
   注释与规范
   关键逻辑加注释，变量命名见名知意
## 第四阶段：扩展功能（锻炼高级能力） 
   IO、复杂逻辑、数据持久化，大幅提升项目完整性。
1. 最高分持久化	用文件保存历史最高分，启动游戏读取，结束游戏更新	Java 文件 IO、数据读写
2. 障碍物系统	地图随机生成障碍物，撞到直接死亡	多物体碰撞检测
3. 多关卡模式	关卡 1：低速无障碍物；关卡 2：高速 + 障碍物	关卡逻辑、难度配置
4. 道具系统	生成加速 / 减速 / 无敌道具，限时生效	道具逻辑、状态时效控制
5. 主题切换	亮色 / 暗色主题，一键切换	配置化、动态绘制
