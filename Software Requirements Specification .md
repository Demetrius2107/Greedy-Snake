
1. Introduction（引言）

1.1 Purpose（目的）

本文档为贪吃蛇游戏（以下简称“本游戏”）的软件需求规格说明书，明确本游戏的开发目标、功能需求、性能需求、界面需求等所有核心需求，作为开发人员编码、测试人员测试、项目验收的唯一标准，同时助力开发过程中思路梳理，确保项目符合预期目标（锻炼Java图形化编程、逻辑实现及代码模块化能力）。

1.2 Scope（范围）

1.2.1 Product Scope（产品范围）

本游戏是一款基于Java Swing开发的图形化单机贪吃蛇游戏，支持单人操作，包含蛇的移动、食物生成、分数统计、游戏状态控制等核心功能，面向个人练手展示及基础娱乐使用，无复杂商业场景需求。

1.2.2 Exclusions（排除范围）

1. 不支持多人联机功能；2. 不支持复杂的网络交互、云存储功能；3. 不涉及付费、广告等商业模块；4. 不支持跨平台适配（仅适配Windows桌面环境，基于Java跨平台特性可兼容常见桌面系统）。

1.3 Definitions, Acronyms and Abbreviations（定义、首字母缩写和缩略语）

1. SRS：Software Requirements Specification，软件需求规格说明书；2. GUI：Graphical User Interface，图形用户界面；3. Swing：Java语言的图形化界面开发工具包；4. Timer：定时器，用于控制蛇的自动移动频率；5. KeyListener：键盘监听器，用于捕捉用户键盘操作。

1.4 References（参考资料）

1. Java Swing开发文档；2. 软件工程需求规格说明书标准模板；3. 贪吃蛇游戏核心逻辑设计思路。

2. Overall Description（总体描述）

2.1 Product Perspective（产品前景）

本游戏为Java练手项目，核心目标是锻炼Java图形化编程、面向对象封装、逻辑算法实现及代码模块化能力，开发一款功能完整、运行流畅的基础版贪吃蛇游戏，同时可在此基础上扩展进阶功能，提升项目完整性。

2.2 Product Functions（产品功能概述）

本游戏核心功能包括：游戏窗口展示、蛇的绘制与自动移动、键盘方向控制、食物随机生成、吃食物增分增长度、游戏结束判定（撞墙、撞自身）、分数实时显示、游戏状态控制（开始、暂停、重新开始），可选扩展功能（难度递增、穿墙模式、最高分持久化等）。

2.3 User Characteristics（用户特征）

本游戏用户为Java学习者（练手展示）及基础娱乐用户，无需专业操作技能，仅需掌握键盘方向键（↑↓←→）、空格（开始/暂停）、R键（重新开始）的基础操作，对计算机操作有基本了解即可。

2.4 Operating Environment（运行环境）

2.4.1 Hardware Environment（硬件环境）

1. 处理器：Intel Core i3及以上；2. 内存：2GB及以上；3. 硬盘：预留100MB及以上存储空间；4. 输入设备：键盘（必备）、鼠标（可选，用于窗口操作）。

2.4.2 Software Environment（软件环境）

1. 操作系统：Windows 10/11（32位/64位）、macOS、Linux（兼容Java运行环境即可）；2. Java运行环境：JDK 1.8及以上；3. 开发工具：IntelliJ IDEA/Eclipse（用于开发调试，用户运行无需安装）。

2.5 Constraints（约束条件）

1. 开发语言：仅使用Java语言，不引入第三方框架，基于原生Swing实现图形化界面；2. 性能约束：游戏运行无明显卡顿（帧率≥30fps），键盘操作响应延迟≤100ms；3. 代码约束：代码需模块化拆分，符合Java编码规范，关键逻辑添加注释，杜绝硬编码。

3. Specific Requirements（具体需求）

3.1 Functional Requirements（功能需求）

3.1.1 Game Window Function（游戏窗口功能）

- 3.1.1.1 窗口创建：启动游戏后，自动生成固定大小的游戏窗口，窗口标题为“Snake Game（贪吃蛇游戏）”。

- 3.1.1.2 窗口控制：支持窗口最小化、关闭操作，关闭窗口时直接终止游戏运行；窗口大小固定，不支持拉伸缩放。

- 3.1.1.3 画板承载：窗口内包含游戏画板，所有游戏元素（蛇、食物、分数、提示文字）均在画板上绘制，无多余冗余组件。

3.1.2 Snake Function（蛇的功能）

- 3.1.2.1 蛇的绘制：蛇由若干个矩形节点组成，蛇头与蛇身体采用不同颜色区分（如蛇头红色、蛇身绿色），初始长度为3个节点，初始位置在窗口中央。

- 3.1.2.2 自动移动：游戏启动后，蛇按照初始方向（如向右）自动匀速移动，移动频率可根据难度调整，每移动一次，蛇头新增一个节点，蛇尾删除一个节点（吃食物时不删除尾部）。

- 3.1.2.3 方向控制：通过键盘↑↓←→键控制蛇的移动方向，禁止直接反向（如当前向上移动时，不能直接按下向下键，避免蛇瞬间撞自身）；按键过快时，需添加方向锁，防止方向错乱。

- 3.1.2.4 长度变化：蛇吃到食物后，长度增加1个节点，分数相应增加，蛇的移动速度可根据分数递增（可选）。

3.1.3 Food Function（食物功能）

- 3.1.3.1 随机生成：游戏启动后，自动在画板内随机生成1个食物（矩形，颜色与蛇区分，如黄色），食物位置不能超出窗口边界，不能与蛇的身体重叠。

- 3.1.3.2 重新生成：蛇吃到食物后，当前食物消失，立即在画板内重新生成一个新的食物，生成规则同上。

3.1.4 Game State Control Function（游戏状态控制功能）

- 3.1.4.1 状态定义：支持4种游戏状态：未开始、运行中、暂停、结束。

- 3.1.4.2 状态切换：按空格键可实现“未开始→运行中”“运行中→暂停”“暂停→运行中”的切换；游戏结束后，按R键可重新开始游戏，重置蛇的位置、长度、分数及食物位置。

- 3.1.4.3 状态提示：不同状态下显示对应提示文字（未开始/暂停：“按空格键开始/继续”；结束：“游戏结束！按R键重新开始”），提示文字位置居中，字体清晰可见。

3.1.5 Score Function（分数功能）

- 3.1.5.1 分数计算：初始分数为0，蛇每吃到一个食物，分数增加10分（可自定义分值），分数实时更新。

- 3.1.5.2 分数显示：在画板顶部居中位置显示当前分数，字体大小适中，颜色与背景区分，确保清晰可见；游戏结束时，显示最终分数。

- 3.1.5.3 最高分持久化（可选）：用本地文件保存历史最高分，游戏启动时读取并显示，游戏结束后若当前分数高于历史最高分，则更新本地文件中的最高分。

3.1.6 Game Over Function（游戏结束功能）

- 3.1.6.1 结束条件：满足以下任一条件，游戏立即结束：①蛇头撞到游戏窗口边界；②蛇头撞到自身身体。

- 3.1.6.2 结束处理：游戏结束后，蛇停止移动，定时器停止工作，显示游戏结束提示文字及最终分数，禁止方向键操作（仅支持R键重新开始、关闭窗口操作）。

3.1.7 Extended Functions（扩展功能，可选）

- 3.1.7.1 难度递增：分数每累计100分，蛇的移动速度加快一次（缩短定时器延迟），可设置3-5个难度等级。

- 3.1.7.2 穿墙模式：可选开启，开启后蛇头撞到边界时，从对面边界穿出，不判定为游戏结束。

- 3.1.7.3 Sound Effect（音效）：吃食物、游戏结束时播放对应音效，支持开启/关闭（可选）。

- 3.1.7.4 Obstacle System（障碍物系统）：地图随机生成固定数量的障碍物（矩形，颜色与蛇、食物区分），蛇头撞到障碍物则游戏结束。

3.2 Non-Functional Requirements（非功能需求）

3.2.1 Performance Requirements（性能需求）

- 3.2.1.1 运行流畅性：游戏运行过程中无卡顿、无闪退，蛇移动、食物生成、分数更新无延迟，帧率稳定在30fps以上。

- 3.2.1.2 响应速度：键盘操作响应延迟≤100ms，按下方向键、空格键、R键后，游戏状态/蛇的方向立即反馈，无明显滞后。

- 3.2.1.3 资源占用：游戏运行时，CPU占用率≤20%，内存占用≤100MB，不影响电脑其他基础操作。

3.2.2 Usability Requirements（易用性需求）

- 3.2.2.1 操作简单：核心操作仅需键盘方向键、空格键、R键，操作逻辑清晰，无需复杂学习成本，用户可快速上手。

- 3.2.2.2 界面友好：游戏界面简洁明了，蛇、食物、分数、提示文字区分清晰，颜色搭配舒适，无刺眼颜色，视觉体验良好。

- 3.2.2.3 容错性：用户误按非核心按键（如字母键、数字键）时，游戏无异常反应，不影响当前游戏状态；误操作后可通过对应按键恢复（如暂停后按空格继续）。

3.2.3 Reliability Requirements（可靠性需求）

- 3.2.3.1 稳定性：游戏连续运行30分钟以上，无闪退、无崩溃、无异常卡顿，数据（分数、蛇的状态）无丢失。

- 3.2.3.2 异常处理：遇到非法操作（如强制关闭窗口、内存不足）时，能正常终止程序，不出现程序卡死、报错弹窗等问题。

3.2.4 Maintainability Requirements（可维护性需求）

- 3.2.4.1 代码模块化：代码按功能拆分不同类（如Main类、GamePanel类、SnakeNode类、Constant类），职责清晰，便于后续修改、扩展。

- 3.2.4.2 代码规范：变量命名、方法命名见名知意，关键逻辑添加注释，符合Java编码规范，便于自己和他人阅读、维护。

- 3.2.4.3 可扩展性：代码设计预留扩展接口，便于后续添加新功能（如道具系统、主题切换），无需大规模修改核心代码。

3.3 Interface Requirements（界面需求）

3.3.1 Overall Interface Design（总体界面设计）

游戏界面简洁，仅包含一个主窗口，无多余弹窗（除必要的提示文字外）；窗口分为两个区域：分数显示区（顶部，高度固定）、游戏绘制区（主体，占窗口大部分面积），无其他冗余组件。

3.3.2 Interface Elements（界面元素要求）

- 3.3.2.1 蛇：蛇头与蛇身体颜色区分明显，矩形节点大小均匀，边缘清晰，移动时无闪烁。

- 3.3.2.2 食物：颜色与蛇、背景区分明显，大小与蛇的节点一致，位置随机，无重叠。

- 3.3.2.3 分数：字体为宋体（或其他清晰字体），字体大小20px左右，颜色为黑色（或与背景对比强烈的颜色），显示在顶部居中位置，实时更新。

- 3.3.2.4 提示文字：字体为宋体，字体大小18px左右，颜色为红色，居中显示，不同游戏状态对应不同提示，清晰易懂。

- 3.3.2.5 背景：背景颜色为浅色（如白色、浅灰色），无复杂图案，避免干扰游戏视线。

3.4 Data Requirements（数据需求）

- 3.4.1 基础数据：蛇的节点坐标（x、y）、蛇的长度、当前移动方向、当前分数、游戏状态、食物坐标。

- 3.4.2 持久化数据（可选）：历史最高分，存储在本地txt文件中，支持读取和更新。

- 3.4.3 数据约束：蛇的坐标、食物坐标需在游戏窗口范围内；分数为非负整数；蛇的长度为正整数（初始为3）。

4. Other Requirements（其他需求）

4.1 Code Requirements（代码需求）

1. 开发语言：Java，JDK版本1.8及以上；2. 开发工具：IntelliJ IDEA/Eclipse；3. 代码需模块化拆分，杜绝所有代码写在一个类中；4. 提取常量类，存储窗口大小、格子大小、颜色、速度等参数，杜绝硬编码；5. 关键逻辑（如碰撞检测、移动算法）添加详细注释。

4.2 Test Requirements（测试需求）

1. 功能测试：所有功能需求均需进行测试，确保每个功能正常运行（如蛇移动、方向控制、吃食物、游戏结束等）；2. 性能测试：测试游戏运行流畅性、响应速度、资源占用，确保符合性能需求；3. 容错测试：测试误操作、异常场景下的程序稳定性。

5. Appendices（附录）

5.1 Appendix A: Core Logic Description（核心逻辑说明）

1. 蛇的移动逻辑：通过Timer定时器，每间隔一定时间（移动速度），获取蛇头当前坐标，根据当前方向计算新的蛇头坐标，新增蛇头节点，若未吃到食物则删除尾部节点；2. 碰撞检测逻辑：蛇头坐标与窗口边界坐标对比（撞墙检测），蛇头坐标与蛇身体其他节点坐标对比（自碰撞检测），蛇头坐标与食物坐标对比（吃食物检测）；3. 随机生成食物逻辑：通过Random类生成随机x、y坐标，判断坐标是否在窗口范围内、是否与蛇身体重叠，若符合条件则确定为食物坐标。

5.2 Appendix B: Key Code Reference（关键代码参考提示）

1. 窗口创建：使用JFrame类创建主窗口，添加JPanel作为游戏画板；2. 绘制功能：重写JPanel的paintComponent方法，绘制蛇、食物、分数、提示文字；3. 定时器：使用javax.swing.Timer控制蛇的自动移动；4. 键盘监听：实现KeyListener接口，捕捉键盘操作，控制蛇的方向及游戏状态。
